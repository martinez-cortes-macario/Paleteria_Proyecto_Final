INTRODUCCIÓN:

La presente página web de paletería fue desarrollada con el propósito de ofrecer a los clientes una manera fácil, rápida y moderna de conocer y adquirir diferentes productos refrescantes y deliciosos. En esta plataforma los usuarios pueden encontrar una gran variedad de paletas, nieves, snacks y aguas frescas, mostrando cada producto con su respectiva imagen y precio para facilitar la elección de compra. El diseño de la página busca brindar una experiencia visual atractiva y sencilla de utilizar para personas de todas las edades.
Además de mostrar el catálogo de productos, la página cuenta con funciones interactivas que permiten mejorar la experiencia del usuario. Entre estas herramientas destacan el botón de carrito y la opción de comprar, las cuales facilitan el proceso de selección y adquisición de productos de manera organizada y práctica. Gracias a estas funciones, los clientes pueden agregar varios artículos al carrito antes de realizar su compra final, simulando el funcionamiento de una tienda en línea moderna.
Otro aspecto importante del sistema es el registro de usuarios. La página incluye una sección donde las personas pueden registrarse proporcionando sus datos básicos para crear una cuenta. Esto permite guardar información del cliente, agilizar futuras compras y ofrecer un servicio más personalizado. El registro también ayuda a mantener un mejor control de pedidos y mejorar la comunicación entre la paletería y sus consumidores.
El objetivo principal de este proyecto es combinar tecnología y comercio para modernizar la venta de productos de una paletería, ofreciendo una alternativa digital cómoda y accesible. De esta manera, los clientes pueden visualizar los productos disponibles, conocer sus precios y realizar compras desde cualquier lugar. Asimismo, la página contribuye a mejorar la organización del negocio y aumentar la satisfacción de los usuarios mediante una plataforma funcional, atractiva y eficiente.

MODELS.PY

from django.db import models


# =========================
# CATEGORÍAS
# =========================
class Categoria(models.Model):
    nombre = models.CharField(max_length=100)

    def __str__(self):
        return self.nombre


# PRODUCTOS
class Product(models.Model):
    name = models.CharField(max_length=100)
    description = models.TextField()
    price = models.DecimalField(max_digits=6, decimal_places=2)

    categoria = models.ForeignKey(
    Categoria,
    on_delete=models.CASCADE,
    related_name='productos',
    null=True,
    blank=True
    )
    imagen = models.ImageField(upload_to='productos/', blank=True, null=True)

    disponible = models.BooleanField(default=True)

    def __str__(self):
        return self.name

# PEDIDOS
class Pedido(models.Model):
    cliente = models.CharField(max_length=100)
    productos = models.ManyToManyField(Product)
    total = models.DecimalField(max_digits=8, decimal_places=2)
    fecha = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"Pedido #{self.id}"



# OPINIONES

class Opinion(models.Model):
    usuario = models.CharField(max_length=100)
    comentario = models.TextField()
    calificacion = models.IntegerField()

    def __str__(self):
        return self.usuario



# PROMOCIONES

class Promocion(models.Model):
    titulo = models.CharField(max_length=100)
    descripcion = models.TextField()
    descuento = models.IntegerField()

    def __str__(self):
        return self.titulo



# INVENTARIO

class Inventario(models.Model):
    producto = models.OneToOneField(Product, on_delete=models.CASCADE)
    cantidad = models.IntegerField()
    ultima_actualizacion = models.DateTimeField(auto_now=True)

    def __str__(self):
        return self.producto.name







Cada modelo representa una tabla dentro de la base de datos, y cada atributo definido en la clase corresponde a una columna de dicha tabla. En este archivo se establecen los tipos de datos que se almacenarán, como textos, números, fechas o imágenes, utilizando campos como CharField, TextField, DecimalField, entre otros. Además, en models.py también se definen las relaciones entre las tablas, como las relaciones de uno a muchos (ForeignKey), muchos a muchos (ManyToManyField) y uno a uno (OneToOneField), lo que permite organizar correctamente la información. Gracias a este archivo, Django puede crear automáticamente las tablas en la base de datos mediante migraciones, facilitando la gestión y manipulación de los datos dentro del sistema. En resumen, models.py es la base sobre la cual se construye toda la lógica de almacenamiento de información en la aplicación.

CATEGORÍA

Tiene como finalidad organizar los productos dentro del sistema mediante clasificaciones específicas. Cuenta con un campo llamado nombre, donde se almacena el tipo de categoría, como por ejemplo paletas, nieves o bebidas. Este modelo es importante porque permite estructurar mejor la información y facilita tanto la búsqueda como la visualización de los productos en la interfaz del usuario. Además, se relaciona con el modelo Product, permitiendo que una categoría pueda contener varios productos. 


PRODUCTS

representa cada uno de los productos disponibles en la paletería. Incluye información detallada como el nombre del producto, una descripción, el precio, una imagen opcional y un campo que indica si el producto está disponible o no. También contiene una relación con el modelo Categoría mediante una llave foránea, lo que permite asignar cada producto a una categoría específica. Este modelo es esencial, ya que es el núcleo del sistema, donde se almacena toda la información necesaria para mostrar los productos a los usuarios y permitir su venta. 

PEDIDOS

Se encarga de registrar las compras realizadas por los clientes. Contiene el nombre del cliente, el total de la compra y la fecha en la que se realizó automáticamente. Además, tiene una relación de muchos a muchos con el modelo Product, lo que significa que un pedido puede incluir varios productos y un mismo producto puede aparecer en distintos pedidos. Este modelo permite llevar un control detallado de las ventas y el historial de compras dentro del sistema.


OPINIONS

Permite almacenar los comentarios y valoraciones de los usuarios sobre los productos o el servicio. Incluye el nombre del usuario, un comentario escrito y una calificación numérica. Este modelo es importante para obtener retroalimentación de los clientes, lo cual puede ayudar a mejorar la calidad del servicio y los productos ofrecidos en la paletería. 








PROMOCIONS

Se utiliza para gestionar las ofertas y descuentos disponibles en el sistema. Contiene un título, una descripción y un valor de descuento que puede representar un porcentaje o cantidad. Su función es permitir mostrar promociones activas a los clientes, lo que ayuda a incentivar las ventas y atraer más usuarios a la aplicación.

INVENTARIO

Tiene como objetivo controlar la cantidad disponible de cada producto en el sistema. Está relacionado con el modelo Product mediante una relación uno a uno, lo que significa que cada producto tiene un único registro de inventario asociado. Incluye la cantidad disponible y la fecha de última actualización, la cual se actualiza automáticamente cada vez que se realiza un cambio. Este modelo es fundamental para evitar problemas de stock y asegurar que no se vendan productos que ya no están disponibles.

FUNCIONAMIENTO COMPLETO 

El funcionamiento del sistema de la paletería se basa en la interacción coordinada entre las URLs, los views, los modelos y los templates dentro del framework Django. Cuando un usuario accede a una página web, por ejemplo al ver los productos o realizar un pedido, primero escribe una dirección en el navegador; esta solicitud es capturada por el sistema de URLs, el cual se encarga de identificar qué vista (view) debe ejecutarse. Una vez que la URL dirige la petición al view correspondiente, este procesa la lógica del sistema, lo que puede incluir consultas, inserciones o actualizaciones en la base de datos mediante los modelos definidos en models.py. Los modelos representan la estructura de la base de datos, como productos, pedidos o inventario, y permiten manipular la información de forma organizada.
Después de obtener o procesar los datos necesarios, el view envía esta información a un template, que es el encargado de la presentación visual. El template utiliza HTML combinado con el sistema de plantillas de Django para mostrar los datos de forma dinámica al usuario, como listas de productos, detalles de pedidos o promociones. Finalmente, el resultado es una interfaz interactiva donde el usuario puede visualizar información, realizar acciones como agregar o eliminar datos, y navegar dentro del sistema. Todo este flujo garantiza que la aplicación funcione de manera estructurada, separando la lógica, los datos y la presentación.
